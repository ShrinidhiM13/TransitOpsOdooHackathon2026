TransitOps Operational Lifecycle Flow

Interactive architectural flowchart, data structures, and state sequences for evaluation

Interactive Pipeline Topology

💻 NextJS Portal Client Frontend (Vercel)
⚡ Express REST API Application Backend
🗄️ MySQL DB Persistent Engine
Description: NextJS communicates directly with the Node Express server. All authentication, role routing, trip allocations, expense reporting, and maintenance triggers are persisted in a multi-table relational MySQL instance.

Stage 1 Draft Created

Fleet Manager creates a trip order. System validates weight matches vehicle load capacity.

Stage 2 Dispatched

Driver license expiry checked. Vehicle and driver status updated to "On Trip". Push notification sent.

Stage 3 En Route / Loading

Driver progress updates trigger UI updates for logistics, updating transit state.

Stage 4 In Transit

Driver executes delivery. Toll logs or fuel expense alerts can be logged on the fly.

Stage 5 Completed

Final odometer validated. Driver/Vehicle set to "Available". Fuel expense registered.

Chronological Offline Syncer

🔌 Connection Lost Actions saved into LocalStorage Sync Queue
Queue[0]: Trip: "In Transit" Queue[1]: Expense: Toll (₹350)
📡 Online Restored Sequential queue processing in background
Sync Mechanism: NextJS captures connection states. While offline, all log modifications and fuel expense entries are cached. Once `navigator.onLine` fires true, the cache is read, pushed via API sequentially, and cleaned upon API validation.

LocalStorage Keys Structure

transitops_pending_expenses [ { id, amount, desc, vehicleId, ... } ]
transitops_pending_status_updates [ { tripId, status, timestamp } ]

VAPID Push Pipeline Flow

1. Manager Dispatch Triggers POST request
2. backend web-push Resolves keys from MySQL DB
3. Service Worker Triggers OS notification
Workflow: When the driver logs in, the portal registers `sw.js` and requests permissions. It gets the subscription object and saves it via `POST /api/push/subscribe`. On dispatch, the server calls `web-push` using these keys to wake the worker and display native alerts.
